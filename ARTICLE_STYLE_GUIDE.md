# 靈元院文章頁固定規格

本文件是靈元院網站文章頁的長期標準。新增或修改任何文章時，除非使用者明確要求改版，否則必須完整沿用本規格與現有 `articles.html`、`articles.js` 的實作，不為單篇文章另做版型。

## 1. 整體版型

- 所有文章統一使用 `articles.html?id=文章ID`。
- 導覽列、下拉選單、頁面背景、內容寬度與首頁保持一致。
- 導覽列包含：首頁、活動、文選、選物、影像、社群、會員登入。
- 文選分類固定為：
  - `spiritual`：靈．修行
  - `worldly`：人．俗世
  - `spirit-world`：異．靈界
  - `reading`：思．讀物
- 手機與桌機共用同一套視覺語言，不得發生文字裁切、圖片重複或按鈕超出畫面。
- 頁面最下方固定使用與首頁相同的靈元院頁尾；頁尾品牌標誌使用 `/assets/footer-logo-gold.svg` 的霧金透明山形 Logo，不再顯示「靈元院 LING YUAN YUAN」文字。
- 文選首頁固定說明文章來源、免費／未來付費閱讀範圍及禁止未經許可轉載。

## 2. 文章內容

- 標題、分類、摘要、正文、圖片皆使用現有文章元件與字體，不在單篇文章加入獨立字型或顏色。
- 中文排版維持目前的襯線正文、霧白文字、深綠背景與霧金重點色。
- 正文保留清楚段落、標題層級與適當行距；避免巨大色塊、巨大標題及過多空白。
- 新文章若由使用者提供原文，原文必須完整保留；新增內容需依使用者指定語氣撰寫。
- 宇色老師文章預設維持沉著、清楚、具生命經驗與自我覺察的語氣。

## 3. 圖片配置

- 需要 AI 圖片時，預設生成三張與文章段落語意直接相關的橫式圖片。
- 第一張作為文章封面，只能出現一次。
- 另外兩張插入正文最適合的段落之間，不集中堆放。
- 不可在正文再次加入封面圖片，避免第一張圖重複。
- 圖片須能正常載入並使用專屬文章資產目錄，例如 `assets/articles/文章ID/`。
- 圖片應明確呈現文章主題，不使用模糊、無關或純裝飾素材。

## 4. 社群支持與全文展開

- 公開內容約為全文前三分之一，使用 `<!-- member-only -->` 分隔前後段。
- 公開網站不顯示會員登入，也不要求一般讀者使用 Google 登入。
- 後段正文保留可辨識的霧化背景，上方顯示約原尺寸三分之二的小型半透明支持提示；背景需可隱約透出。
- 固定提示為：
  - 文章未完，繼續閱讀
  - 若這篇文章對你有所啟發，歡迎訂閱
  - YouTube、追蹤 Facebook，持續收到新的靈修解析。
  - 訂閱 靈元院YouTube
  - 追蹤 靈元院 Facebook
  - 繼續閱讀全文
- YouTube 固定連到 `https://www.youtube.com/@lyyuan03`。
- Facebook 固定連到 `https://www.facebook.com/share/18zfvhPkBF/?mibextid=wwXIfr`。
- 社群支持屬於邀請，不強迫驗證；讀者可直接按「繼續閱讀全文」在原頁展開後段。
- 不再使用手機系統分享選單，也不得把 Google 登入重新放回公開導覽列。

## 5. 延伸閱讀

- 每篇文章正文結束後固定顯示小型按鈕：
  - `延伸閱讀｜宇色靈修著作`
- 固定連結：
  - `https://lyyuan.tw/books.html?v=spiritual-books-20260703-refresh`
- 按鈕靠右、尺寸小、框線柔和、使用霧金色，不製作大型宣傳色塊。

## 6. 社群與分享按鈕

- 不顯示「社群」文字，只顯示圖示。
- 圖示列置於延伸閱讀按鈕下方、文章頁尾上方，靠右排列。
- 按鈕固定為 36px 圓形，圖示約 18px，方便年長讀者辨識與點擊。
- 顏色保留平台識別，但統一柔化以符合靈元院 CI：
  - 透明度約 `0.76`
  - 飽和度約 `0.58`
  - 亮度約 `0.88`
  - 無明顯陰影
  - 使用淡色細框
  - 滑入時才稍微提高亮度
- 固定順序與功能：
  1. Facebook：直接前往靈元院官方帳號 `https://www.facebook.com/share/18zfvhPkBF/?mibextid=wwXIfr`
  2. Instagram：直接前往靈元院官方帳號 `https://www.instagram.com/lyyuan03/`
  3. LINE：分享目前文章標題與網址
  4. Telegram：分享目前文章標題與網址
  5. Email：開啟郵件 App，預填主旨與文章網址
  6. 複製連結：複製目前文章網址
- Facebook、Instagram 是「前往官方帳號」；LINE、Telegram、Email 是「分享文章」。不得再次改用手機系統分享選單。

## 7. 無障礙與操作

- 每個圖示必須有正確的 `aria-label` 與 `title`。
- 色彩不可暗到看不清，也不可鮮豔到搶過正文。
- 點擊範圍不可因視覺縮小而過小。
- 手機版須保持一列可見；空間不足時優先縮小間距，不縮小到難以點擊。
- 所有外部連結使用安全的新分頁設定。

## 8. 發布檢查

每篇新文章發布前必須確認：

- 分類正確。
- 封面沒有重複。
- 三張圖片皆能載入。
- 公開文字約為前三分之一。
- 前三分之一後的霧化支持提示正常。
- 「繼續閱讀全文」能在原頁展開後段。
- 延伸閱讀連結正確。
- 六個社群／分享圖示順序、顏色與功能一致。
- 桌機及手機皆無裁切、重疊與過大空白。
- 頁尾與首頁一致。

## 9. 經營者後台

- 經營者固定由 `https://lyyuan.tw/admin.html` 進入文章後台。
- 後台使用指定管理員 Gmail 登入；目前管理員為 `lyyuan03@gmail.com`。
- 後台入口不放在公開導覽列，經營者自行收藏網址。
- Firebase Firestore 與 Storage 的管理員規則不得放寬。
- 一般訪客即使知道後台網址，也不得取得文章新增、修改、刪除或圖片上傳權限。
- 後台固定提供「匯出全部文章」，下載 ZIP 內含 Firestore 與 GitHub 靜態文章的 JSON、Markdown、文章索引 CSV、圖片網址清單與搬遷說明；匯出不得修改原資料。

以上規格適用於往後所有靈元院文選文章。


## 文章與社群的品牌層級

- 官網以「靈元院」為主品牌；導覽選單的影像與社群子目錄，固定分為「靈元院官方」與「宇色老師」兩組。
- 每組先顯示品牌標題，再列出平台，避免四個帳號看起來屬於同一層級。
- 頁尾只保留靈元院官方帳號，不放宇色老師個人帳號。
- 文章閱讀提示框的主要按鈕只引導靈元院 YouTube 與 Facebook；宇色老師以較小的「影音／文章」文字連結呈現，不得搶過主按鈕。
- 靈元院負責院務、法儀、活動與官方發布；宇色老師負責靈修解析、生命觀點、影音與個人文章。


## Facebook 與社群連結預覽

- 每篇可分享文章必須有獨立的靜態分享頁，路徑固定為 `article/文章-slug.html`，並在文章資料加入 `sharePath`。
- 靜態分享頁的 HTML `<head>` 必須直接寫入 `og:type`、`og:site_name`、`og:title`、`og:description`、`og:url`、`og:image` 與 Twitter Card 標記；不得只依賴 JavaScript 或 Firebase 動態填入。
- `og:image` 使用該篇文章第一張圖的完整 HTTPS 網址。建議另備 1200 × 630 像素社群圖；若未另製，沿用文章首圖。
- Facebook 分享按鈕固定使用 `https://www.facebook.com/sharer/sharer.php?u=網址編碼後的分享頁網址`，開啟使用者自己的貼文編輯器。
- LINE、Telegram、Email 與複製連結均使用同一個 `sharePath`，確保各平台讀取相同標題與首圖。
- 新增文章時，發布流程必須同步建立分享頁；Facebook 已快取過的網址需透過 Sharing Debugger 重新抓取。


## 文章互動統計

- 每篇文選文章都必須依文章 ID 統計「閱讀」、「分享」與「複製」三項數據；新文章發布後自動沿用，不需逐篇另寫程式。
- 「閱讀」以同一瀏覽器工作階段、同一篇文章計算一次，重新整理頁面不得反覆灌入點閱。
- 「分享」在讀者按下 Facebook、LINE、Telegram 或 Email 分享功能時累計一次；前往官方 Facebook／Instagram 帳號不列入文章分享。
- 「複製」只在文章分享網址成功複製到剪貼簿後累計一次。
- 文選文章頁與文章列表皆顯示三項數據，樣式須小型、柔和並符合霧金 CI，不得搶過文章標題與正文。
- 經營者後台固定提供「文章互動統計」，列出所有動態與靜態文章的閱讀、分享、複製次數，預設依閱讀數排序。
- 統計資料固定儲存在 Firebase Firestore 的 `articleMetrics` collection，文件 ID 使用文章 ID；不得把數據只存在單一裝置或 HTML。
- 統計失敗不得阻斷文章載入、分享或複製功能。
